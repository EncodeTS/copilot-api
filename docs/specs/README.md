# Local specifications

This directory is the repository's local issue tracker for durable specifications.

- One specification per `<feature-slug>.md` file.
- Put `Status: <triage-role>` and `Type: spec` near the top.
- When a specification is decomposed, place one ticket per file under `<feature-slug>/issues/`.
- Keep private session IDs, credentials, local paths, and user conversation content out of committed fixtures and specifications.
